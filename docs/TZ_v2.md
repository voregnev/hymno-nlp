# Техническое задание v2.0

## 1. Цель и назначение

Разработка платформы для сравнительного лингвистического, богословского и литургического анализа гимнографических текстов православного богослужения на трёх языках:
- Церковнославянский (ЦСЯ, синодальная редакция)
- Греческий (Byzantine/Medieval, Παρακλητική Venice 1858)
- Русский (опционально, Alpha+)

## 2. Корпус

| Источник | Язык | Объём | Приоритет |
|---|---|---|---|
| Октоих глас 8 | ЦСЯ + греч. | ~800 ед. | **P0 — MVP** |
| Октоих гласы 1–7 | ЦСЯ + греч. | ~5 600 ед. | P1 — Alpha |
| Минея: Пасха, Рождество, Богоявление | ЦСЯ + греч. | ~1 200 ед. | P1 — Alpha |
| Типикон (структурная разметка) | ЦСЯ | ~1 200 стр. | P2 — Beta |
| Минея месячная (все 12 тт.) | ЦСЯ + греч. | ~15 000 стр. | P3 — Prod |
| НЗ (eBible grcbyz + cu) | ЦСЯ + греч. | ~31K стихов | **P0 — bootstrap** |
| UD OCS-PROIEL | OCS | 198K токенов | **P0 — NLP bootstrap** |

## 3. Архитектура

См. `docs/architecture.md` для полной схемы.

**Три хранилища с единым ID:**
- `Neo4j` — граф структуры и связей (без full_text)
- `PostgreSQL` — тексты, строфы, аннотации
- `Qdrant` — векторный поиск (BGE-M3, dense + sparse)

## 4. Онтология атрибутов

См. `docs/data_model.md` для полного списка.

**Атомарная единица:** `:TextUnit` — тропарь / ирмос / стихира / кондак / седален / богородичен / светилен.

**Место в богослужении** — 5-мерная координата:
```
book / office / section / [glas + day_of_week] / [ode_number] / unit_type + order
```

## 5. ML-пайплайн

| Этап | Инструмент | Задача |
|---|---|---|
| Scraping | httpx + BeautifulSoup | azbyka.ru → raw text |
| Парсинг (L1) | regex | Структурные атрибуты |
| Парсинг (L2) | алгоритм | Акростих |
| Парсинг (L3) | Claude API | Автор, жанр (fallback) |
| Классификация | SetFit + XLM-R | Fine-tuning жанров |
| Выравнивание | Bertalign + LaBSE | ЦСЯ ↔ греч. пары |
| Эмбеддинги | BGE-M3 | Векторы для поиска |
| Активное обучение | small-text | Расширение разметки |

## 6. Функциональные требования

### 6.1 Поиск
- Полнотекстовый (tsvector, морфология)
- Семантический (векторный, BGE-M3)
- Гибридный (BM25 + dense, RRF fusion)
- Фильтры: глас, жанр, язык, автор, праздник, раздел службы

### 6.2 Сравнение
- Side-by-side: ЦСЯ | Греческий | (опц.) Русский
- Diff-подсветка расхождений
- Similarity score
- Аннотации экспертов

### 6.3 Навигация
- По структуре богослужения (через Neo4j граф)
- По литургическому календарю
- По автору / подобну / гласу

### 6.4 Аналитика (Beta+)
- Конкорданс (KWiC)
- Частотный анализ лексики
- Граф-визуализация связей
- Экспорт TEI XML

## 10. Ключевые риски

1. **Нет готового гимнографического корпуса** → oцифровка = 40% трудозатрат. Митигация: azbyka.ru закрывает всё без OCR.
2. **ЦСЯ — low-resource язык** → нет LLM, субтокенизация неоптимальна. Митигация: кросс-языковой трансфер с русского (XLM-R/LaBSE).
3. **Свободные переводы** → не все ЦСЯ тексты имеют 1:1 греческий оригинал. Митигация: Bertalign нативно поддерживает 1-many.
4. **Вариативность редакций** → синодальная / студийская / иерусалимская расходятся. Митигация: атрибут `edition` + граф HAS_VARIANT.
5. **Поэтика не автоматизируется** → метр, образный ряд требуют ручного анализа. Митигация: отложить до Beta, экспертные аннотации.
